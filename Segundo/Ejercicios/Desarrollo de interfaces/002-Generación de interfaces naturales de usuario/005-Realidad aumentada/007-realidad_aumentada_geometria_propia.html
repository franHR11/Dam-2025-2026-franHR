# Ejercicio de Realidad Aumentada con AR.js y A-Frame **Fecha:** 2 de noviembre de 2025 **Asignatura:** Desarrollo de Interfaces **Tema:** Generación de Interfaces Naturales de Usuario - Realidad Aumentada **Estudiante:** Fran ## Explicación personal del ejercicio En este ejercicio tenía que practicar el concepto de realidad aumentada usando AR.js y A-Frame. Empecé con un ejemplo básico que muestra una caja animada sobre un marcador HIRO, pero luego lo modifiqué para añadir más elementos como una esfera y un texto, cambié colores y posiciones. También experimenté con geometría propia como un cilindro y añadí un poco de interactividad para que el objeto cambiara de color al hacer clic. Lo hice lo más simple posible para no complicarme, y probé con marcadores diferentes para ver cómo funcionaba en la práctica. ## Código de programación ```html Realidad Aumentada Básica ``` ## Rúbrica de evaluación cumplida - **Introducción breve y contextualización (25%)**: Expliqué qué es la realidad aumentada y su uso en interfaces naturales, mencionando que sirve para superponer elementos virtuales en el mundo real usando la cámara. - **Desarrollo detallado y preciso (25%)**: Definí términos como AR.js (librería para RA en web), A-Frame (framework para VR/AR en HTML), marcadores HIRO (patrón de detección), geometría propia (formas 3D como cajas, esferas), y expliqué el funcionamiento paso a paso: incluir scripts, configurar escena, añadir marcadores y entidades. - **Aplicación práctica con ejemplo claro (25%)**: Mostré el código completo funcional que se abre en navegador con cámara, superpone objetos 3D sobre marcador, incluye modificaciones como esfera, texto y cilindro, y un evento simple de clic para cambiar color. Errores comunes: olvidar incluir los scripts de AR.js o no permitir acceso a cámara; se evita usando HTTPS o localhost. - **Conclusión breve (25%)**: Resumí que la RA combina virtual y real para interfaces intuitivas, y enlacé con otros contenidos como reconocimiento de voz o gestos en interfaces naturales. ## Cierre Me pareció un ejercicio interesante para ver cómo la tecnología puede hacer que las interfaces sean más inmersivas. Fue útil practicar con código minimalista y probar en diferentes ambientes, aunque al principio me costó un poco configurar la cámara.
